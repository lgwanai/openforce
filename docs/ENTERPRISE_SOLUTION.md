## 一、产品概述

OpenForce 是新一代企业级 AI 多智能体协作平台，采用独创的"三省六部"架构设计，通过分层治理、职责分离和智能编排，为企业提供安全、可控、高效的 AI 自动化解决方案。

### 核心价值主张

**让 AI 像组织一样运作，而非像工具一样执行。**

传统 AI Agent 系统面临三大困境：
- **职责混乱**：单一 Agent 承担过多职责，导致决策质量下降
- **不可控**：AI 自主行动缺乏监督，存在安全与合规风险
- **资源失控**：任务执行缺乏预算约束，成本难以预测

OpenForce 通过借鉴国家治理机制，构建了一套完整的 AI 组织架构，让 AI 系统具备：
- 清晰的权责边界
- 完善的监督机制
- 可预测的资源消耗

```

└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                      尚书省（编排调度）                    │
│   • 任务拆解与 DAG 编排                                   │
│   • 资源调度与并发控制                                    │
│   • 异常处理与重规划                                      │
└─────────────────────┬───────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┬─────────────┐
        ▼             ▼             ▼             ▼
   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
   │  兵部   │  │  户部   │  │  工部   │  │  刑部   │
   │代码执行 │  │信息研究 │  │环境管理 │  │代码审查 │
   └─────────┘  └─────────┘  └─────────┘  └─────────┘
        │             │             │             │
        └─────────────┴─────────────┴─────────────┘
                              │
                      ┌───────▼───────┐
                      │    都察院     │
                      │  安全与审计   │
                      └───────────────┘
```

### 2.2 四级响应机制

| 级别 | 响应时间 | 处理方式 | 适用场景 |
|------|----------|----------|----------|
| L0 | <1s | 中书省直接回复 | 简单问答、寒暄 |
| L1 | <3s | 中书省调用轻量工具 | 文件读取、目录列表 |
| L2 | <10s | 委派户部调研 | 信息查询、知识问答 |
| L3 | 按需 | 尚书省编排多部门协作 | 复杂开发任务、运维自动化 |

**业务价值**：80% 的日常请求在 L0/L1 级别完成，大幅降低计算成本和响应延迟。

## 三、核心能力

### 3.1 智能上下文管理

**问题**：传统 Agent 在长对话中容易"失忆"，重复查询相同信息。

**OpenForce 方案**：
- 会话级上下文复用，避免重复工具调用
- 相对时间自动转换（"明天" → "2024-04-07"）
- 关键信息自动提取与存储

**效果**：减少 40% 以上的冗余 API 调用，提升用户体验。

### 3.2 完善的安全体系

#### 人机协同审批（Human-in-the-Loop）

```
高风险操作 → 生成一次性审批令牌 → 用户确认 → 执行前快照比对 → 执行
```

**防伪机制**：
- 包含 `task_id + action_hash + nonce` 的签名令牌
- 执行前对象快照比对，防止 TOCTOU 攻击
- 令牌一次性使用，防止重放攻击

#### 沙箱隔离

- 所有文件操作限制在沙箱目录
- 真实路径归一化，防止路径穿越
- 禁止符号链接，防止逃逸

#### SSRF 防护

- 阻止访问内网 IP 和 localhost
- 阻止非 HTTP 协议（file://、ftp://）
- 不跟随重定向

### 3.3 资源预算控制

| 维度 | 控制机制 | 作用 |
|------|----------|------|
| Token 预算 | 每任务配额 | 防止单个任务消耗过多 Token |
| 时间预算 | 超时熔断 | 防止死循环和无限等待 |
| 并发屏障 | 部分成功释放 | 防止并行任务死锁 |

**业务价值**：AI 运行成本可预测、可控制，告别"账单惊喜"。

### 3.4 企业级可靠性

- **原子性 Token 消费**：预算扣减使用数据库原子操作
- **状态持久化**：任务状态实时保存，支持断点续传
- **异常熔断**：指数退避重试，防止雪崩

## 四、典型应用场景

### 4.1 智能研发助手

```
用户: 帮我开发一个用户认证模块

中书省: 识别为复杂开发任务，转尚书省
尚书省: 拆解为：环境搭建→代码编写→单元测试→代码审查
工部: 创建 Python 虚拟环境，安装依赖
兵部: 编写认证模块代码
刑部: 运行测试，检查代码质量
都察院: 敏感操作（安装依赖）触发人工审批
→ 最终交付完整的认证模块
```

**价值**：研发效率提升 30%，代码质量有保障。

### 4.2 运维自动化

```
用户: 检查服务器日志，找出异常并生成报告

户部: 搜索相关日志分析方法
兵部: 执行日志分析脚本
都察院: 检测到敏感数据访问，触发审批
用户确认后: 生成安全报告
```

**价值**：7x24 自动化运维，人工介入最小化。

### 4.3 知识库问答

```
用户: 公司的差旅报销流程是什么？

户部: 搜索内部知识库
→ 返回准确的流程说明和表单链接
```

**价值**：员工自助服务，释放 HR 行政精力。

### 4.4 数据分析与报告

```
用户: 生成上季度销售分析报告

户部: 收集销售数据
兵部: 执行数据分析脚本
礼部: 生成标准化报告文档
```

**价值**：自动化报告生成，决策支持更及时。

## 五、技术优势

### 5.1 多模型支持

| Agent 角色 | 推荐模型 | 备选模型 |
|------------|----------|----------|
| 中书省（决策） | Claude 3.5 Sonnet | GPT-4o |
| 户部（研究） | Claude 3.5 Sonnet | DeepSeek V3 |
| 兵部（代码） | Claude 3.5 Sonnet | GPT-4o |
| 刑部（审查） | DeepSeek V3 | GPT-4o-mini |

**价值**：根据任务类型选择最优模型，成本效益最大化。

### 5.2 语义路由

- 不依赖硬编码的 `if-else`
- 基于意图理解动态选择执行路径
- 自适应复杂度，简单任务快速响应

### 5.3 ReAct 自验证

每个子任务执行后：
1. 检查执行结果
2. 验证是否满足验收标准
3. 不通过则自动修正重试
