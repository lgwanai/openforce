# SOP: 架构设计

## 适用场景
- 新系统设计
- 架构评审
- 技术选型

## 拆解模板
1. **[architect] 需求分析**: 理解业务需求、非功能需求（性能/安全/扩展）
2. **[architect] 模块划分**: 边界上下文、服务拆分、接口定义
3. **[db_architect] 数据设计**: 数据库 Schema、索引策略、分库分表
4. **[architect] 安全边界**: 认证方案、授权模型、数据分类、网络策略
5. **[doc_writer] 文档输出**: 架构决策记录 (ADR)、接口规范、部署图

## 依赖关系
```
Task1 → Task2 → Task3 (并行)
Task1 → Task2 → Task4 (并行)
Task2,3,4 → Task5 (文档)
```
