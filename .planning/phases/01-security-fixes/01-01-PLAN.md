## Current Vulnerability

**File:** `src/tools/base.py:143-183`

**Issue:** Uses `shell=True` with user-controlled input, enabling command injection.

```python
# VULNERABLE CODE:
full_cmd = f"mkdir -p /tmp/Library/Caches && ln -sf ... && {executable} {command}"
result = subprocess.run(full_cmd, shell=True, ...)
```

**Attack vector:** If `command` contains `; rm -rf /`, it executes arbitrary code.

## Research Solution

From RESEARCH.md Pattern 1:
- Never use `shell=True`
- Use argument lists with explicit executable paths
- Implement whitelist validation before execution
